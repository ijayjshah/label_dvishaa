You need to develop a fully dynamic, production-ready website exactly according to the provided Figma design and database structure.

## Core Requirements

### 1. UI & Responsiveness

* Follow the Figma design strictly for all pages, spacing, typography, colors, layouts, and interactions.
* The entire website must be fully responsive:

  * Mobile-first approach
  * Tablet responsive
  * Desktop responsive
* Admin panel must also be mobile-friendly.
* Mobile UI should not be limited to the current Figma layout only. Improve the mobile experience by adding better usability, optimized sections, improved navigation, and additional mobile-friendly layouts wherever necessary while keeping the design consistent.

---

## 2. Authentication & User Management

### Signup Flow

At signup, collect only:

* Full Name
* Email
* Phone Number

### Login

* Secure authentication system
* JWT/session-based authentication
* Maintain complete customer history after login

### User Data

When placing an order, collect:

* Full Address
* City
* State
* Pincode
* Additional order-related details

All customer activities must be stored and visible in the admin panel:

* Orders
* Gallery uploads
* Profile details
* Payment history
* User activity

---

## 3. Database & Storage

### Database

Use:

* PostgreSQL

### Image Storage

Use:

* Cloudinary

Reason:

* Avoid server load
* Better image optimization and CDN delivery

All uploaded images should go directly to Cloudinary.

---

## 4. Website Sections

### Dynamic Sections

These sections must be fully manageable from the admin panel:

* Categories
* Products
* Homepage banners
* Gallery
* Orders
* Users
* Mobile section content

### Static Sections

These sections do NOT require APIs:

* About Us
* Contact Us

## 5. Categories

* Currently display only 4 main categories on the frontend.
* No subcategories required for now.
* Categories should remain fully dynamic from admin panel.
* Admin can:

  * Add category
  * Edit category
  * Delete category
  * Upload category image

---

## 6. Product Management & Size System

Each product must support multiple sizes.

### Product Upload Requirements

While uploading a product, admin should be able to define:

* Product name
* Description
* Price
* Images
* Category
* Available sizes
* Stock quantity
* Product status

### Size Management

* Sizes should be dynamic.
* Admin can select multiple available sizes for each product.
* Example:

  * S
  * M
  * L
  * XL
  * XXL

### Product View Page

On the product detail/view page:

* Display available sizes clearly
* User must select size before adding to cart or buying
* Show selected size in cart and order details

Sizes should also be manageable from admin panel for future scalability.

---

Create a gallery section where:

* Users can upload images

* Admin can:

  * View uploads
  * Delete uploads
  * Moderate content

Gallery should be optimized for mobile devices.
